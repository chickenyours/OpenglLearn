1. 模块作用

ApplicationModule 是应用程序模块，负责：

  • 窗口管理：创建和管理 GLFW 窗口
  • OpenGL 上下文管理：创建 OpenGL 上下文，支持上下文交接给渲染线程
  • GLAD 加载器：提供 OpenGL 函数加载器
  • 事件处理：处理窗口事件

2. 与 Render 模块的协作

Application 模块与 Render 模块通过OpenGL 上下文交接机制协作：

主线程 渲染线程 | | | 1. ApplicationModule.Startup() | | - 创建窗口 | | - 创建 OpenGL 上下文 | | | | 2. RenderModule. | | InitializeFromApplication() | | - 获取上下文指针 | | - 传递给 RenderThread | | | | 3. ApplicationModule. | | ReleaseGLContext() | | - 释放主线程上下文 | | |----------> | | 4. RenderThread.ThreadMain() | | - glfwMakeContextCurrent() | | - 上下文成为渲染线程当前上下文 | | - 执行 OpenGL API

3. 基本用法

3.1 创建并启动 Application 模块

#include "Application/public/application_module.h"

Application::ApplicationModule appModule;

// 配置窗口参数
Application::ApplicationConfig config;
config.width = 800;
config.height = 600;
config.title = "OpenGL Window";
config.majorVersion = 4;
config.minorVersion = 6;
appModule.SetConfig(config);

// 启动模块
if (!appModule.Startup()) {
    std::cerr << "Failed to start ApplicationModule!" << std::endl;
    return -1;
}

3.2 获取窗口和 OpenGL 上下文

// 获取窗口指针
GLFWwindow* window = appModule.GetWindow();

// 获取 OpenGL 上下文指针（用于交接给渲染线程）
void* glContext = appModule.GetGLContext();

// 获取 OpenGL 函数加载器
auto loader = appModule.GetGLProcAddressLoader();

3.3 上下文交接（给 Render 模块使用）

#include "Application/public/application_module.h"
#include "Render/Public/render_module.h"

Application::ApplicationModule appModule;
// ... 配置并启动 appModule ...

Render::RenderModule renderModule;

// InitializeFromApplication 会自动完成上下文交接
if (!renderModule.InitializeFromApplication(appModule)) {
    std::cerr << "Failed to initialize RenderModule!" << std::endl;
    return -1;
}

// 此时：
// 1. RenderModule 已获取 OpenGL 上下文指针
// 2. 主线程已释放 OpenGL 上下文
// 3. 渲染线程已获取 OpenGL 上下文

3.4 手动上下文交接

如果需要手动控制上下文交接过程：

// 1. 获取上下文指针
void* glContext = appModule.GetGLContext();
GLFWwindow* window = appModule.GetWindow();

// 2. 将上下文传递给渲染线程
renderThread->SetGLContext(window, glContext);

// 3. 释放主线程的 OpenGL 上下文
appModule.ReleaseGLContext();

// 4. 启动渲染线程
renderThread->Start();
// 渲染线程会在 ThreadMain 中调用 glfwMakeContextCurrent()

4. API 详细说明

4.1 配置接口

方法 说明
SetConfig(const ApplicationConfig& config) 设置窗口配置（尺寸、标题、OpenGL 版本等）

4.2 ApplicationConfig 结构

struct ApplicationConfig {
    int width = 800;              // 窗口宽度
    int height = 600;             // 窗口高度
    const char* title = "Window"; // 窗口标题
    int majorVersion = 4;         // OpenGL 主版本
    int minorVersion = 6;         // OpenGL 次版本
};

4.3 生命周期接口

方法 说明
Startup() 启动模块，创建窗口和 OpenGL 上下文
Shutdown() 关闭模块，释放窗口和上下文
IsStarted() 检查模块是否已启动

4.4 上下文交接接口

方法 说明
GetGLContext() 获取 OpenGL 上下文指针（返回窗口指针，GLFW 上下文与窗口绑定）
ReleaseGLContext() 释放当前线程的 OpenGL 上下文，调用后当前线程不能再使用 OpenGL
MakeGLContextCurrent(window, context) 静态方法，在指定线程使 OpenGL 上下文成为当前上下文

4.5 访问接口

方法 说明
GetWindow() 获取 GLFW 窗口指针
GetGLProcAddressLoader() 获取 OpenGL 函数加载器（用于 GLAD）

5. 完整示例

5.1 与 Render 模块配合使用

#include "Application/public/application_module.h"
#include "Render/Public/render_module.h"

int main() {
    // 1. 创建并配置 Application 模块
    Application::ApplicationModule appModule;
    Application::ApplicationConfig config;
    config.width = 800;
    config.height = 600;
    config.title = "OpenGL Application";
    config.majorVersion = 4;
    config.minorVersion = 6;
    appModule.SetConfig(config);

    // 2. 启动 Application 模块
    if (!appModule.Startup()) {
        return -1;
    }

    // 3. 创建并初始化 Render 模块（自动完成上下文交接）
    Render::RenderModule renderModule;
    if (!renderModule.InitializeFromApplication(appModule)) {
        appModule.Shutdown();
        return -1;
    }

    // 4. 获取 RHI 设备进行渲染操作
    auto* device = renderModule.GetDevice();
    // ... 使用 device 进行渲染 ...

    // 5. 清理
    renderModule.Shutdown();
    appModule.Shutdown();
    return 0;
}

5.2 手动上下文交接

#include "Application/public/application_module.h"
#include "Render/Private/Runtime/render_thread.h"

int main() {
    Application::ApplicationModule appModule;
    // ... 配置并启动 ...

    // 1. 创建渲染线程
    Render::RHI::RenderThread renderThread;

    // 2. 获取上下文和窗口
    void* glContext = appModule.GetGLContext();
    GLFWwindow* window = appModule.GetWindow();

    // 3. 传递给渲染线程
    renderThread.SetGLContext(window, glContext);

    // 4. 释放主线程上下文
    appModule.ReleaseGLContext();

    // 5. 启动渲染线程
    renderThread.Start();

    // ... 使用渲染线程 ...

    // 6. 清理
    renderThread.Stop();
    appModule.Shutdown();
    return 0;
}

6. 注意事项

重要提示：
  1. 上下文交接后主线程不能使用 OpenGL：调用 ReleaseGLContext() 后，主线程不能再执行任何 OpenGL 调用
  2. 渲染线程独占 OpenGL 上下文：所有 OpenGL API 调用必须在渲染线程执行
  3. 窗口销毁前必须先停止渲染线程：确保渲染线程已释放上下文
  4. GLFW 函数不是线程安全的：不要在多个线程同时调用 GLFW 函数
推荐做法：
  • 使用 RenderModule::InitializeFromApplication() 自动完成上下文交接
  • 不要手动调用 glfwMakeContextCurrent()，让 RenderThread 管理
  • 主线程只负责业务逻辑，渲染操作全部交给渲染线程

7. 常见问题

Q1: 为什么主线程不能直接使用 OpenGL？

OpenGL 上下文是线程绑定的，一个上下文只能在一个线程中使用。为了将渲染操作集中到渲染线程，主线程需要释放上下文，让渲染线程独占。

Q2: 如何验证上下文交接成功？

可以在渲染线程中调用 glGetString(GL_VERSION) 验证 OpenGL 上下文是否可用。

Q3: ReleaseGLContext() 后还能重新获取上下文吗？

可以，通过 MakeGLContextCurrent(window, context) 可以在任意线程重新使上下文成为当前上下文。
