1. 模块作用

RenderModule 是 Module/Render 模块的核心入口类，它：

  • 继承自 IModule 接口，可被应用程序或 ECS 统一管理
  • 管理图形库平台状态（窗口、上下文、函数加载器）
  • 负责 OpenGL 函数指针的加载
  • 在启动时检查环境合法性
  • 提供 RHI 设备创建入口

2. 架构概述

Render 模块采用前端命令代理 + 专职渲染线程 + 委托后门的架构：

  • RHI 前端层：给上层提供统一接口，负责分配句柄、组装命令、投递到队列
  • 命令层：统一表示所有 RHI 操作，保证顺序执行
  • 后端执行层：在渲染线程中将命令翻译成 OpenGL API 调用
  • 渲染线程：专职渲染线程，持有图形上下文，批量执行命令

详细架构说明请参阅 Render 模块架构文档。

3. 启动前置条件

在调用 RenderModule::Startup() 之前，应用程序必须确保：

  1. 窗口系统已初始化（如 GLFW、SDL 等）
  2. OpenGL Context 已创建并成为当前线程的活动上下文
  3. 已准备好 OpenGL 函数地址获取函数（如 glfwGetProcAddress）

4. OpenGL Loader 设置方法

模块使用 glad 作为 OpenGL 函数加载器。应用程序需要通过 SetGLProcAddressLoader() 传入一个函数指针获取器。

4.1 函数签名

using GLProcAddressLoader = void* (*)(const char* name);

4.2 常见平台的 Loader

平台 Loader 函数
GLFW glfwGetProcAddress
SDL SDL_GL_GetProcAddress
Windows/WGL 自定义加载函数

5. Startup / Shutdown 用法

5.1 基本用法

#include "Render/Public/render_module.h"

Render::RenderModule renderModule;

// 1. 设置平台状态
renderModule.SetPlatformReady(true);

// 2. 设置上下文状态
renderModule.SetContextReady(true);

// 3. 设置 OpenGL 函数加载器
renderModule.SetGLProcAddressLoader(&glfwGetProcAddress);

// 4. 启动模块
if (!renderModule.Startup()) {
    // 处理启动失败
    return -1;
}

// ... 使用模块 ...

// 5. 关闭模块
renderModule.Shutdown();

5.2 状态检查

// 检查是否已启动
if (renderModule.IsStarted()) {
    // 模块已就绪
}

// 获取当前平台状态
auto state = renderModule.GetPlatformState();
switch (state) {
    case Render::GraphicsLibraryPlatformState::Ready:
        // 完全就绪
        break;
    case Render::GraphicsLibraryPlatformState::Error:
        // 发生错误
        break;
    // ... 其他状态
}

6. 与应用程序配合方式

应用程序负责创建窗口和 OpenGL 上下文，然后将其传递给 RenderModule。

6.1 完整示例（使用 ApplicationModule）

#include "Application/public/application_module.h"
#include "Render/Public/render_module.h"

int main() {
    // 1. 创建并启动 Application 模块
    Application::ApplicationModule appModule;
    Application::ApplicationConfig config;
    config.width = 800;
    config.height = 600;
    config.title = "OpenGL Window";
    config.majorVersion = 4;
    config.minorVersion = 6;
    appModule.SetConfig(config);

    if (!appModule.Startup()) {
        return -1;
    }

    // 2. 创建并初始化 Render 模块
    Render::RenderModule renderModule;
    if (!renderModule.InitializeFromApplication(appModule)) {
        return -1;
    }

    // 3. 获取 RHI 设备进行渲染
    auto* device = renderModule.GetDevice();
    // ... 渲染逻辑 ...

    // 4. 清理
    renderModule.Shutdown();
    appModule.Shutdown();
    return 0;
}

7. RHI 前端接口使用

RHI::RHIFrontend 是渲染模块的前端接口，提供CPU 端资源加载和命令代理模式的资源操作。

RHI 架构职责划分：
  • RHIFrontend（前端）：负责 CPU 端操作（加载纹理资产、命令投递）
  • Device（后端）：负责 GPU 端操作（创建 GPU 资源、上传数据）
  • RenderThread（渲染线程）：专职执行 OpenGL API，所有图形调用在此线程执行

7.1 获取 RHIFrontend 和 Device

#include "Render/Public/RHI/rhi_frontend.h"
#include "Render/Public/RHI/RHI_device.h"

// 从 RenderModule 获取后端设备和渲染线程
auto* device = renderModule.GetDevice();
auto* renderThread = renderModule.GetRenderThread();

// 创建前端接口，使用 RenderModule 的渲染线程
Render::RHI::RHIFrontend frontend(renderThread);

// 设置前端使用的后端设备
frontend.SetDevice(device);

7.2 加载纹理资产（CPU 端操作）

#include "Render/Public/RHI/RHI_texture.h"

// 使用 RHI 前端加载纹理资产（CPU 端操作）
// 此方法从文件加载图像数据到 CPU 内存，不创建 GPU 资源
auto asset = frontend.LoadTextureFromFile("path/to/texture.png", false, true);
if (!asset) {
    std::cerr << "Failed to load texture asset!" << std::endl;
    return;
}

// 获取纹理描述
const auto& desc = asset->GetDesc();
std::cout << "Width: " << desc.width << std::endl;
std::cout << "Height: " << desc.height << std::endl;

7.3 创建 GPU 纹理资源（GPU 端操作）

// 使用 RHIFrontend 投递创建命令到渲染线程
// 此方法投递命令到创建队列，不立即执行
auto texHandle = frontend.CreateEmptyTexture(desc);

// 手动唤醒渲染线程执行命令
renderThread->Wakeup();

// 等待渲染线程执行完成
std::this_thread::sleep_for(std::chrono::milliseconds(100));
命令执行流程：
  1. 主线程调用 CreateEmptyTexture()
  2. 命令进入创建队列
  3. 主线程调用 Wakeup() 唤醒渲染线程
  4. 渲染线程批量取出命令并执行

7.4 创建 Buffer（命令投递模式）

#include "Render/Public/RHI/RHI_buffer.h"

using namespace Render::RHI;

// 定义顶点结构
struct Vertex {
    float position[2];
    float color[3];
};

// 顶点数据
Vertex vertices[] = {
    {{-0.5f, -0.5f}, {1.0f, 0.0f, 0.0f}},
    {{ 0.5f, -0.5f}, {0.0f, 1.0f, 0.0f}},
    {{ 0.0f,  0.5f}, {0.0f, 0.0f, 1.0f}}
};

// 创建 Buffer 描述
BufferDesc desc;
desc.size = sizeof(vertices);
desc.stride = sizeof(Vertex);
desc.usage = static_cast(BufferUsage::Vertex);
desc.memoryUsage = MemoryUsage::CpuToGpu;
desc.cpuAccess = static_cast(CpuAccessMode::Write);

// 创建 Buffer（返回句柄，命令投递到渲染线程）
BufferHandle handle = frontend.CreateBuffer(desc, vertices, sizeof(vertices));

7.5 委托后门

#include 

// 使用委托后门执行 OpenGL 调用（在渲染线程执行）
frontend.EnqueueDelegate([](Render::RHI::BackendDelegateContext& ctx) {

7.6 停止渲染线程

frontend.Stop();

8. 与 ECS 配合方式

ECS 系统不应直接依赖 OpenGL 具体类型，而应通过 RenderModule 或 RHI::Device 进行交互。

8.1 推荐模式

// ECS 系统持有 RenderModule 的引用或指针
class RenderSystem {
public:
    RenderSystem(Render::RenderModule& renderModule)
        : renderModule_(renderModule) {}

    void Update() {
        auto* device = renderModule_.GetDevice();
        if (device) {
            // 通过 RHI 接口进行渲染操作
            // 不直接依赖 GLDevice、GLTexture 等具体类型
        }
    }

private:
    Render::RenderModule& renderModule_;
};

8.2 ECS 集成步骤

  1. 应用程序创建并启动 ApplicationModule
  2. 应用程序创建并启动 RenderModule（通过 InitializeFromApplication）
  3. ECS 调度系统通过模块对象获取 Device
  4. ECS 系统只调用抽象接口，不依赖具体 OpenGL 类型

9. 示例代码

9.1 最小可用示例

#include "Render/Public/render_module.h"

// 假设已有 OpenGL 上下文和 loader
extern void* MyGLLoader(const char* name);

int main() {
    Render::RenderModule module;

    module.SetPlatformReady(true);
    module.SetContextReady(true);
    module.SetGLProcAddressLoader(&MyGLLoader);

    if (module.Startup()) {
        // 获取设备并进行渲染
        auto* device = module.GetDevice();
        // ...
        module.Shutdown();
    }

    return 0;
}

9.2 状态机示例

Render::RenderModule module;

// 逐步设置状态
module.SetPlatformReady(true);
assert(module.GetPlatformState() == Render::GraphicsLibraryPlatformState::PlatformReady);

module.SetContextReady(true);
assert(module.GetPlatformState() == Render::GraphicsLibraryPlatformState::ContextReady);

module.SetGLProcAddressLoader(&glfwGetProcAddress);

// 启动后进入 Ready 状态
if (module.Startup()) {
    assert(module.GetPlatformState() == Render::GraphicsLibraryPlatformState::Ready);
}

10. 平台状态说明

状态 说明
Uninitialized 未初始化
PlatformReady 窗口系统已就绪
ContextReady OpenGL 上下文已就绪
FunctionLoaderReady 函数加载器已就绪
Ready 模块完全就绪
Error 发生错误

11. 注意事项

重要提示：
  1. 不要重复调用 Startup()：模块会检测 started_ 状态，重复调用会直接返回 true
  2. Shutdown() 可安全重复调用：内部有状态检查
  3. 确保 Context 存在：glad 加载必须在有效的 OpenGL 上下文中进行
  4. 错误处理：Startup() 失败时返回 false，不会抛出异常
  5. 推荐依赖 ApplicationModule：使用 InitializeFromApplication() 方法可以自动处理所有前置条件
  6. 线程安全：RHI 前端接口是线程安全的，但 OpenGL 调用只在渲染线程执行

12. 常见问题

Q1: 为什么 GetDevice() 返回 nullptr？

可能原因：

  • RenderModule 未正确启动
  • OpenGL 函数加载失败（gladLoadGLLoader 返回 false）
  • 平台状态未设置正确（需要先设置 PlatformReady 和 ContextReady）

Q2: 可以在主线程直接调用 OpenGL 吗？

可以，但需要确保 OpenGL 上下文在当前线程激活。推荐使用 RHI 前端接口或委托后门，这样可以确保在渲染线程执行。

Q3: 如何添加新的图形 API 后端？

实现 IRHIBackendExecutor 接口，并在渲染线程中使用新的执行器。详细架构请参阅 Render 模块架构文档。
