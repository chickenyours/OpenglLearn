📖 1. 模块简介

ResourceManager 模块是一个基于模块化的资源管理系统，提供了完整的资源生命周期管理功能：

  • 资源引用计数自动管理
  • 延迟加载与按需加载
  • 资源缓存与复用
  • 自定义资源释放策略
  • 线程安全的资源池

🏗️ 3. 核心类说明

3.1 ILoadable（可加载接口）

所有资源的基类接口，定义了资源的基本行为：

方法 说明
IsLoad() 检查资源是否已加载
Release() 释放资源（纯虚函数，需子类实现）

3.2 ILoadFromConfig（配置文件加载接口）

继承自 ILoadable，支持从配置文件加载资源：

class ILoadFromConfig : public ILoadable {
public:
    virtual bool LoadFromConfigFile(const std::string& configFile) = nullptr;
};

3.3 ResourceHandle<T>（资源句柄）

智能句柄类，自动管理资源的生命周期：

  • 拷贝构造时自动增加引用计数
  • 析构时自动减少引用计数
  • 引用计数为 0 时自动释放资源
  • 支持移动语义

3.4 LoadOptions<T>（加载选项）

配置资源加载行为的选项类：

成员 说明
key 资源的唯一标识键
generator 资源生成器函数
onZeroCallFunc 引用计数为 0 时的回调函数
onGetRestoreCallFunc 获取已存在资源时的回调函数

3.5 ResourceManagerModule（资源管理器模块）

核心管理器类，提供全局资源管理功能：

  • 单例模式访问
  • 集成到模块管理系统
  • 模板化的资源池管理

🚀 4. 使用指南

4.1 获取 ResourceManager 实例

auto* resourceManager = manager.GetModule<Resource::ResourceManagerModule>();

4.2 加载资源（四种方式）

方式一：FromGenerator（使用生成器）

auto handle = resourceManager->Get<Resource::ILoadable>(
    Resource::FromGenerator<Resource::ILoadable>(
        "my_resource",
        []() -> std::unique_ptr<Resource::ILoadable> {
            return std::make_unique<MyResource>();
        }
    )
);

方式二：FromConfig（从配置文件加载）

auto handle = resourceManager->Get<MyConfigResource>(
    Resource::FromConfig<MyConfigResource>(
        "config/my_resource.json",
        nullptr,  // onZero 回调
        nullptr   // onRestore 回调
    )
);

方式三：FromPointer（从已有指针加载）

auto ptr = std::make_unique<MyResource>();
auto handle = resourceManager->Get<Resource::ILoadable>(
    Resource::FromPointer<Resource::ILoadable>(
        "existing_resource",
        std::move(ptr)
    )
);

方式四：FromKey（仅获取已存在的资源）

auto handle = resourceManager->Get<Resource::ILoadable>(
    Resource::FromKey<Resource::ILoadable>("existing_key")
);

💡 6. 完整示例

// 1. 定义自定义资源类
class TextureResource : public Resource::ILoadFromConfig {
public:
    GLuint textureID = nullptr;
    
    bool LoadFromConfigFile(const std::string& configFile) override {
        // 从配置文件加载纹理
        glGenTextures(1, &textureID);
        // ... 加载纹理数据
        isLoad_ = true;
        return true;
    }
    
    void Release() override {
        if (textureID) {
            glDeleteTextures(1, &textureID);
            isLoad_ = false;
        }
    }
};

// 2. 在模块系统中使用
int main() {
    auto& manager = ModuleManager::Instance();
    manager.Startup();
    
    // 获取 ResourceManager
    auto* resourceManager = manager.GetModule<Resource::ResourceManagerModule>();
    
    // 加载纹理资源
    auto textureHandle = resourceManager->Get<TextureResource>(
        Resource::FromConfig<TextureResource>("assets/texture.json")
    );
    
    // 使用资源
    if (textureHandle && textureHandle->IsLoad()) {
        glBindTexture(GL_TEXTURE_2D, textureHandle->textureID);
        // ... 渲染
    }
    
    // 句柄析构时自动释放资源
    manager.Shutdown();
    return 0;
}
💡 提示： 资源句柄采用引用计数管理，多个句柄可以共享同一资源。当最后一个句柄被销毁时，资源会自动释放。

❓ 7. 常见问题

Q1: 如何自定义资源释放策略？

使用 FromGenerator 或 FromConfig 的 onZeroCallFunc 参数：

auto handle = resourceManager->Get<MyResource>(
    Resource::FromGenerator<MyResource>(
        "my_resource",
        []() { return std::make_unique<MyResource>(); },
        [](const std::string& key, MyResource* res) {
            // 自定义释放逻辑
            std::cout << "Resource " << key << " will be released" << std::endl;
        }
    )
);

Q2: 资源缓存如何工作？

当使用相同的 key 请求资源时，ResourceManager 会返回已缓存的资源实例，而不是重新创建。通过 onGetRestoreCallFunc 可以感知资源复用：

auto handle1 = resourceManager->Get<MyResource>(
    Resource::FromGenerator<MyResource>(
        "shared_resource",
        []() { return std::make_unique<MyResource>(); },
        nullptr,
        [](const std::string& key, MyResource* res) {
            std::cout << "Resource restored: " << key << std::endl;
        }
    )
);

// 第二次获取相同 key 的资源会触发 onGetRestoreCallFunc
auto handle2 = resourceManager->Get<MyResource>(
    Resource::FromKey<MyResource>("shared_resource")
);

Q3: 线程安全吗？

是的，ResourceManager 内部使用互斥锁保护资源池操作，支持多线程安全访问。

⚠️ 注意： 虽然资源池是线程安全的，但资源对象本身的操作需要用户自行保证线程安全。
