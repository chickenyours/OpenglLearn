RHI 纹理验证器使用文档

文档版本：1.0
最后更新：2026 年 4 月 11 日
适用模块：Render Module (OpenGL 4.6+)

1. 模块概述

RHI 纹理验证器子系统提供了一种解耦的、可扩展的纹理验证机制，允许用户通过委托函数验证纹理加载是否成功，而无需直接依赖后端具体类型。

主要优势：
  • 解耦设计：测试代码不需要包含后端头文件（如 gl_texture.h），也不需要直接使用 dynamic_cast 转换为具体后端类型
  • 可扩展：支持注册多个验证器，同时验证多个后端
  • 委托模式：使用 std::function 实现灵活的验证逻辑，支持 Lambda 表达式
  • 跨平台：不同图形 API 后端可以注册各自的验证器
  • 统一接口：所有验证器使用相同的接口，便于切换和比较

2. 核心类型

2.1 TextureValidationResult 结构

struct TextureValidationResult {
    bool isValid = false;           // 验证是否通过
    uint32_t width = 0;             // 纹理宽度
    uint32_t height = 0;            // 纹理高度
    uint32_t mipLevels = 0;         // Mipmap 层级数
    std::size_t nativeHandle = 0;   // 后端原生句柄（如 OpenGL Texture ID）
    std::string errorMessage;       // 错误信息（验证失败时）
};
成员 类型 说明
isValid bool 验证是否通过，true 表示验证成功
width uint32_t 纹理宽度（从后端查询的实际值）
height uint32_t 纹理高度（从后端查询的实际值）
mipLevels uint32_t Mipmap 层级数
nativeHandle std::size_t 后端原生句柄，如 OpenGL 的 Texture ID
errorMessage std::string 验证失败时的错误描述

2.2 TextureValidator 委托类型

using TextureValidator =
    std::function<TextureValidationResult(const TextureAsset&)>;

验证器是一个接受 TextureAsset 常量引用并返回 TextureValidationResult 的可调用对象。

3. 基本用法

3.1 注册验证器

#include "Render/Public/RHI/rhi_texture_validator.h"
#include "Render/Private/Backend/Opengl/gl_texture_validator.h"

int main() {
    // 注册 OpenGL 纹理验证器
    Render::Backend::OpenGL::RegisterOpenGLTextureValidator();

    // ... 后续使用
    return 0;
}

3.2 使用指定验证器验证纹理

#include "Render/Public/RHI/rhi_texture_validator.h"

using namespace Render::RHI;

// 加载纹理
auto texture = device->LoadTexture2D("path/to/texture.png", true);
if (!texture) {
    std::cerr << "Failed to load texture!" << std::endl;
    return;
}

// 使用 OpenGL 验证器验证
auto result = ValidateTexture(*texture, "OpenGL");

if (result.isValid) {
    std::cout << "Validation passed!" << std::endl;
    std::cout << "Width: " << result.width << std::endl;
    std::cout << "Height: " << result.height << std::endl;
    std::cout << "Native Handle: 0x"
              << std::hex << result.nativeHandle << std::dec << std::endl;
} else {
    std::cerr << "Validation failed: " << result.errorMessage << std::endl;
}

3.3 使用所有已注册验证器

// 运行所有已注册的验证器
auto allResults = ValidateTexture(*texture);

for (const auto& [name, validatorResult] : allResults) {
    std::cout << "[" << name << "] "
              << (validatorResult.isValid ? "PASSED" : "FAILED") << std::endl;
    if (!validatorResult.isValid) {
        std::cout << "    Error: " << validatorResult.errorMessage << std::endl;
    }
}

3.4 自定义委托验证器

#include "Render/Public/RHI/rhi_texture_validator.h"

using namespace Render::RHI;

// 定义自定义验证逻辑（使用 Lambda 表达式）
TextureValidator customValidator =
    [](const TextureAsset& tex) -> TextureValidationResult {
    TextureValidationResult result;

    // 检查纹理资产是否有效
    if (!tex.Check()) {
        result.errorMessage = "Texture asset check failed";
        return result;
    }

    // 检查是否有有效的格式
    const auto& desc = tex.GetDesc();
    if (desc.format == Format::Unknown) {
        result.errorMessage = "Texture format is Unknown";
        return result;
    }

    // 检查尺寸是否大于 0
    if (desc.width == 0 || desc.height == 0) {
        result.errorMessage = "Texture dimensions are zero";
        return result;
    }

    result.isValid = true;
    result.width = desc.width;
    result.height = desc.height;
    result.mipLevels = desc.mipLevels;
    return result;
};

// 使用自定义验证器
auto customResult = customValidator(*texture);
std::cout << "Custom validator: "
          << (customResult.isValid ? "PASSED" : "FAILED") << std::endl;

4. 完整测试程序示例

以下示例展示如何在主线程中使用 RHI 加载纹理，并使用委托函数验证：

// RHI 纹理验证器测试程序
#include <iostream>
#include <functional>

#include "application_module.h"
#include "render_module.h"
#include "Render/Public/RHI/RHI_device.h"
#include "Render/Public/RHI/RHI_texture.h"
#include "Render/Public/RHI/rhi_texture_validator.h"
#include "Render/Private/Backend/Opengl/gl_texture_validator.h"

using namespace Render::RHI;

int main() {
    std::cout << "=== RHI Texture Validator Test ===" << std::endl;

    // ========== 1. 创建并启动 Application 模块 ==========
    Application::ApplicationModule appModule;
    Application::ApplicationConfig config;
    config.width = 800;
    config.height = 600;
    config.title = "RHI Texture Validator Test";
    config.majorVersion = 4;
    config.minorVersion = 6;
    appModule.SetConfig(config);

    std::cout << "[AppModule] Creating window and OpenGL context..." << std::endl;
    if (!appModule.Startup()) {
        std::cerr << "[AppModule] Failed to startup ApplicationModule!" << std::endl;
        return -1;
    }
    std::cout << "[AppModule] ApplicationModule started successfully!" << std::endl;

    // ========== 2. 创建并初始化 Render 模块 ==========
    Render::RenderModule renderModule;
    std::cout << "[RenderModule] Initializing from ApplicationModule..." << std::endl;
    if (!renderModule.InitializeFromApplication(appModule)) {
        std::cerr << "[RenderModule] Failed to initialize RenderModule!" << std::endl;
        appModule.Shutdown();
        return -1;
    }
    std::cout << "[RenderModule] RenderModule started successfully!" << std::endl;

    // ========== 3. 注册 OpenGL 纹理验证器 ==========
    std::cout << "[Validator] Registering OpenGL texture validator..." << std::endl;
    Render::Backend::OpenGL::RegisterOpenGLTextureValidator();

    // ========== 4. 获取 RHI 设备并加载纹理 ==========
    auto* device = renderModule.GetDevice();
    if (!device) {
        std::cerr << "[RHI] Failed to get RHI device!" << std::endl;
        renderModule.Shutdown();
        appModule.Shutdown();
        return -1;
    }

    std::string texturePath = "Module/Test/tex.png";
    std::cout << "[RHI] Loading texture from: " << texturePath << std::endl;
    auto texture = device->LoadTexture2D(texturePath, true);
    if (!texture) {
        std::cerr << "[RHI] Failed to load texture!" << std::endl;
        renderModule.Shutdown();
        appModule.Shutdown();
        return -1;
    }
    std::cout << "[RHI] Texture loaded successfully!" << std::endl;

    // 输出纹理参数
    const auto& desc = texture->GetDesc();
    std::cout << "\n=== Texture Parameters ===" << std::endl;
    std::cout << "Width: " << desc.width << " pixels" << std::endl;
    std::cout << "Height: " << desc.height << " pixels" << std::endl;
    std::cout << "Mip Levels: " << desc.mipLevels << std::endl;

    // ========== 5. 使用委托函数验证纹理 ==========
    std::cout << "\n[Validator] Validating texture..." << std::endl;
    auto result = ValidateTexture(*texture, "OpenGL");

    std::cout << "\n=== Validation Result ===" << std::endl;
    std::cout << "Validator: OpenGL" << std::endl;
    std::cout << "Is Valid: " << (result.isValid ? "YES" : "NO") << std::endl;

    if (result.isValid) {
        std::cout << "Width: " << result.width << std::endl;
        std::cout << "Height: " << result.height << std::endl;
        std::cout << "Native Handle: 0x"
                  << std::hex << result.nativeHandle << std::dec << std::endl;
        std::cout << "\n[SUCCESS] RHI Texture Loading Test PASSED!" << std::endl;
    } else {
        std::cerr << "Error: " << result.errorMessage << std::endl;
        std::cerr << "\n[FAILED] RHI Texture Loading Test FAILED!" << std::endl;
    }

    // ========== 6. 清理 ==========
    renderModule.Shutdown();
    appModule.Shutdown();

    return result.isValid ? 0 : 1;
}

7. API 参考

7.1 注册验证器

void RegisterTextureValidator(
    const std::string& name,
    TextureValidator validator
);
参数 说明
name 验证器名称，用于后续查找（如 "OpenGL"）
validator 验证函数（可调用对象）

7.2 获取验证器

TextureValidator GetTextureValidator(
    const std::string& name
);
参数 说明
name 验证器名称
返回值 验证器函数，如果不存在则返回空函数

7.3 验证纹理（单个验证器）

TextureValidationResult ValidateTexture(
    const TextureAsset& texture,
    const std::string& validatorName
);
参数 说明
texture 要验证的纹理对象
validatorName 验证器名称
返回值 验证结果

7.4 验证纹理（所有验证器）

std::vector<std::pair<std::string, TextureValidationResult>>
ValidateTexture(const TextureAsset& texture);
参数 说明
texture 要验证的纹理对象
返回值 所有验证器的结果列表，每项为 (名称，结果) 对

8. 注意事项

重要提示：
  1. OpenGL 上下文：验证 OpenGL 纹理前必须确保 OpenGL 上下文已激活，否则 glGetTextureLevelParameteriv 等调用会失败
  2. 验证器注册时机：验证器应在使用前注册，通常在程序启动时或首次使用前
  3. 线程安全：验证器注册是线程安全的（内部使用互斥锁），但验证操作本身不是线程安全的
  4. 验证器生命周期：注册的验证器会一直存在，直到程序结束
  5. 工作目录：运行测试程序时请确保工作目录正确，以便找到纹理文件（通常在 bin 目录下运行）

9. 扩展说明

未来可扩展的验证器类型：

  • Vulkan 验证器：使用 vkGetImageSubresourceLayout 验证
  • D3D11/D3D12 验证器：使用 ID3D11DeviceContext::Map 验证
  • Metal 验证器：使用 MTLTexture 的 descriptor 验证
  • 性能分析验证器：验证纹理显存占用、带宽等性能指标
  • 质量验证器：分析纹理压缩质量、mipmap 完整性等

10. 解耦设计说明

10.1 原始设计的问题

在原始设计中，测试程序需要：

  • 直接 #include <GLFW/glfw3.h> 和 #include <glad/glad.h>
  • 直接 #include "Render/Private/Backend/Opengl/gl_texture.h"
  • 使用 dynamic_cast<GLTexture*> 转换为具体后端类型
  • 直接调用 glTexture->GetHandle() 获取 OpenGL 句柄

10.2 委托模式的优势

使用委托模式后：

  • 测试代码只依赖 RHI 公共接口：不需要包含后端头文件
  • 验证逻辑封装在委托函数中：后端细节对测试代码透明
  • 易于切换到其他图形 API 后端：只需注册不同的验证器
  • 支持自定义验证逻辑：用户可以定义自己的验证器
最佳实践：
  • 测试代码只包含 RHI 公共头文件（如 rhi_texture_validator.h）
  • 验证器实现在后端目录中，由后端模块负责维护
  • 使用 RegisterOpenGLTextureValidator() 等函数注册验证器
  • 通过 ValidateTexture() 统一接口进行验证
  • 使用 Lambda 表达式定义临时的自定义验证逻辑

12. 常见问题

Q1: 为什么验证器返回的宽高为 0？

确保在验证前 OpenGL 上下文已激活。如果在没有有效上下文的情况下调用 glGetTextureLevelParameteriv，查询会失败。

Q2: 如何注册多个验证器？

多次调用 RegisterTextureValidator() 即可，每个验证器使用不同的名称：

RegisterTextureValidator("OpenGL", ValidateOpenGLTexture);
RegisterTextureValidator("Custom", myCustomValidator);

Q3: 验证器会影响性能吗？

验证器主要用于调试和测试，不建议在生产环境的渲染循环中频繁调用。OpenGL 验证器会调用 glGetTextureLevelParameteriv，这是一个相对昂贵的操作。
