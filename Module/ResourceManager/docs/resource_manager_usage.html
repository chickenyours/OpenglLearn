ResourceManager 模块使用文档

1. 模块概述

ResourceManager 模块是项目资源管理系统的核心，提供：

  • 资源生命周期管理（加载、引用计数、释放）
  • 资源池化存储
  • 灵活的加载策略（从文件、从指针、从生成器）
  • 自定义释放回调

2. 文件结构

重构后的 ResourceManager 模块采用分层设计，文件职责清晰：

2.1 公共头文件 (Public/)

文件 职责
resource_loadable.h 基础接口：ILoadable, ILoadFromConfig
resource_handle.h 资源句柄声明：ResourceHandle<T>

resource_load_options.h 加载选项和工厂函数声明

resource_manager_module.h 资源管理器类声明

2.2 私有实现 (Private/)

文件 职责
resource_manager_module.cpp 非模板实现：GetInstance(), Startup(), Shutdown()

3. 核心类说明

3.1 ILoadable（资源基础接口）

namespace Resource {

class ILoadable {
public:
    virtual ~ILoadable() = default;
    
    // 检查资源是否已加载
    bool IsLoad() const;
    
    // 释放资源（纯虚函数）
    virtual void Release() = 0;

protected:
    bool isLoad_ = false;
};

}

3.2 ILoadFromConfig（配置文件加载接口）

namespace Resource {

class ILoadFromConfig : public ILoadable {
public:
    // 从配置文件加载资源
    virtual bool LoadFromConfigFile(const std::string& configFile) = 0;
};

}

3.3 ResourceHandle<T>（资源句柄）

namespace Resource {

template<typename T>
class ResourceHandle {
public:
    ResourceHandle() = default;
    ResourceHandle(std::nullptr_t);
    ResourceHandle(const ResourceHandle& other);
    ResourceHandle(ResourceHandle&& other) noexcept;
    ResourceHandle& operator=(const ResourceHandle& other);
    ResourceHandle& operator=(ResourceHandle&& other) noexcept;

    T* operator->();
    T& operator*();
    T* get() const;
    explicit operator bool() const;
    const std::string& GetName() const;
};

}
注意：ResourceHandle 的拷贝构造和拷贝赋值有特殊语义——会通过 ResourceManagerModule 重新登记引用，不是普通的值语义复制。

3.4 LoadOptions<T>（加载选项）

namespace Resource {

template<typename T>
struct LoadOptions {
    std::string key;
    std::function<std::unique_ptr<T>()> generator;
    std::function<void(const std::string&, T*)> onZeroCallFunc;
    std::function<void(const std::string&, T*)> onGetRestoreCallFunc;
};

}

4. 工厂函数

4.1 FromConfig（从配置文件加载）

// 适用于实现 ILoadFromConfig 接口的资源类
auto options = Resource::FromConfig<MyResource>("config.json");
auto handle = Resource::ResourceManagerModule::GetInstance().Get(options);

4.2 FromPointer（从已有指针）

auto ptr = std::make_unique<MyResource>();
auto options = Resource::FromPointer<MyResource>("my_key", std::move(ptr));
auto handle = Resource::ResourceManagerModule::GetInstance().Get(options);

4.3 FromGenerator（从生成器函数）

auto options = Resource::FromGenerator<MyResource>(
    "my_key",
    []() -> std::unique_ptr<MyResource> {
        return std::make_unique<MyResource>();
    }
);
auto handle = Resource::ResourceManagerModule::GetInstance().Get(options);

4.4 FromKey（从键名恢复）

// 用于从已有资源恢复句柄（不创建新资源）
auto options = Resource::FromKey<MyResource>("my_key");
auto handle = Resource::ResourceManagerModule::GetInstance().Get(options);

5. 基本用法

5.1 获取模块实例

#include "resource_manager_module.h"

auto& resourceManager = Resource::ResourceManagerModule::GetInstance();

5.2 加载资源

// 方式 1：从配置文件加载
auto textureHandle = resourceManager.Get(
    Resource::FromConfig<Texture2D>("texture.json")
);

// 方式 2：从生成器加载
auto modelHandle = resourceManager.Get(
    Resource::FromGenerator<Model>(
        "my_model",
        []() -> std::unique_ptr<Model> {
            return std::make_unique<Model>();
        }
    )
);

// 方式 3：从已有指针加载
auto shaderPtr = std::make_unique<Shader>();
auto shaderHandle = resourceManager.Get(
    Resource::FromPointer<Shader>("my_shader", std::move(shaderPtr))
);

5.3 使用资源

if (textureHandle) {
    // 使用 operator-> 访问资源
    textureHandle->Bind();
    
    // 或使用 operator*
    Texture2D& tex = *textureHandle;
    tex.Bind();
}

5.4 自定义释放策略

auto handle = resourceManager.Get(
    Resource::FromGenerator<MyResource>(
        "my_resource",
        []() { return std::make_unique<MyResource>(); },
        // 零引用时回调（可选）
        [](const std::string& key, MyResource* res) {
            std::cout << "Resource " << key << " released" << std::endl;
        },
        // 再次获取时恢复回调（可选）
        [](const std::string& key, MyResource* res) {
            std::cout << "Resource " << key << " restored" << std::endl;
        }
    )
);

6. 完整示例

6.1 最小可用示例

#include "resource_manager_module.h"
#include "resource_loadable.h"

namespace Resource {

// 定义一个测试资源类
struct TestResource : public ILoadable {
    int value = 0;

    void Release() override {
        isLoad_ = false;
    }

    void SetLoaded(bool loaded) {
        isLoad_ = loaded;
    }
};

} // namespace Resource

int main() {
    // 1. 获取 ResourceManager 实例
    auto& resourceManager = Resource::ResourceManagerModule::GetInstance();

    // 2. 启动模块
    resourceManager.Startup();

    // 3. 加载资源
    auto handle = resourceManager.Get(
        Resource::FromGenerator<Resource::ILoadable>(
            "test_resource",
            []() -> std::unique_ptr<Resource::ILoadable> {
                auto ptr = new Resource::TestResource();
                ptr->value = 42;
                ptr->SetLoaded(true);
                return std::unique_ptr<Resource::ILoadable>(ptr);
            }
        )
    );

    // 4. 使用资源
    if (handle) {
        std::cout << "Resource value: " << handle->value << std::endl;
    }

    // 5. 关闭模块
    resourceManager.Shutdown();

    return 0;
}

6.2 在模块系统中使用

#include "module_manager.h"
#include "resource_manager_module.h"

int main() {
    // 1. 获取并启动 ModuleManager
    auto& manager = ModuleManager::Instance();

        return -1;
    }

    // 2. 获取 ResourceManager 模块
    auto* resourceManager = manager.GetModule<
        Resource::ResourceManagerModule

    if (!resourceManager) {
        return -2;
    }

    // 3. 使用 ResourceManager 加载资源
    auto handle = resourceManager->Get(
        Resource::FromConfig<Texture2D>("texture.json")
    );

    // 4. 使用资源...

    // 5. 关闭模块系统
    manager.Shutdown();

    return 0;
}

7. 依赖关系

推荐的文件包含顺序（从底层到高层）：

  1. resource_loadable.h - 最底层，无外部依赖
  2. resource_load_options.h - 依赖 resource_loadable.h
  3. resource_handle.h - 轻量声明，无重量级依赖
  4. resource_manager_module.h - 聚合所有头文件
提示：大多数情况下只需包含 resource_manager_module.h，它会自动包含所有必需的头文件。

8. 注意事项

1. 句柄拷贝语义：ResourceHandle 的拷贝构造和拷贝赋值会通过 ResourceManagerModule 重新登记引用，这不是普通的值语义复制。
2. 资源释放策略：默认情况下，资源引用计数归零时会自动释放。可通过 onZeroCallFunc 自定义释放行为。
3. 线程安全：当前 ResourcePool 内部使用 std::mutex 保护，但外部调用仍需注意线程安全问题。
4. 模板可见性：由于模板实现在 .inl 文件中，确保 resource_manager_module.h 正确包含了 .inl 文件。

9. 常见问题

Q1: 为什么 Get<T>() 返回空句柄？

可能原因：

  • generator 返回了 nullptr
  • FromConfig 加载失败
  • 资源类型 T 未正确实现 ILoadable 接口

Q2: 如何自定义资源释放行为？

使用 FromGenerator/FromPointer 的 onZeroCallFunc 参数：

auto handle = resourceManager.Get(
    Resource::FromGenerator<MyResource>(
        "key",
        generator,
        [](const std::string& key, MyResource* res) {
            // 自定义释放逻辑
        }
    )
);

Q3: 资源何时会被真正释放？

当最后一个 ResourceHandle 析构（或重置）时，引用计数归零，触发释放回调。默认行为是调用资源的 Release() 方法并从池中移除。

11. 总结

ResourceManager 模块提供了灵活的资源管理机制，通过 ResourceHandle 实现自动引用计数，通过工厂函数支持多种加载方式。重构后的文件结构更加清晰，便于维护和扩展。
