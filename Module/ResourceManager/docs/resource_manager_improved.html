ResourceManager 模块改进版使用文档

改进完成！ ResourceManager 现已支持任意类型的资源管理，不再要求资源类型继承特定接口。

1. 改进概述

本次改进的核心目标是取消"资源必须继承 ILoadable"的要求，使 ResourceManager 能够管理任意类型的资源。

1.1 改进前的问题

  • 资源对象被迫继承框架接口（ILoadable / ILoadFromConfig）
  • 纯数据对象、第三方对象、已有对象无法自然接入资源系统
  • 资源对象本身与资源池管理策略耦合过深

1.2 改进后的优势

  • 资源对象保持纯对象 - 无需继承任何接口
  • 资源管理器负责生命周期 - 引用计数、销毁策略由管理器控制
  • 支持多来源资源接入 - 工厂创建、外部模块创建、RHI 层创建等

2. 核心设计

2.1 ControlBlock - 资源控制块

ControlBlock 是新设计的核心，负责管理资源的生命周期：

  • 持有资源的唯一所有权（通过 std::unique_ptr）
  • 管理引用计数（std::atomic）
  • 执行销毁函数和零引用回调
template<typename T>
class ControlBlock {
    std::unique_ptr<T> resource_;      // 资源所有权
    DestroyFunc destroyFunc_;            // 销毁函数
    ZeroRefCallback zeroRefCallback_;    // 零引用回调
    std::atomic<size_t> refCount_;      // 引用计数
};

2.2 ResourceHandle - 资源句柄

ResourceHandle 是用户访问资源的句柄，通过共享 ControlBlock 实现引用计数：

  • 拷贝构造/拷贝赋值时增加引用计数
  • 析构时减少引用计数
  • 提供 operator-> 和 operator* 访问资源
template<typename T>
class ResourceHandle {
    std::string name_;
    std::shared_ptr<ControlBlock<T>> controlBlock_;  // 共享控制块
};

3. 文件结构

文件 职责
resource_control_block.h/.inl 资源控制块 - 管理引用计数和销毁策略
resource_handle.h/.inl 资源句柄 - 用户访问资源的接口

4. 基本用法

4.1 简单资源类型（无需继承）

// 定义一个普通结构体，无需继承任何接口
struct DataResource {
    int id;
    std::string name;
    float value;
    
    DataResource(int i, const std::string& n, float v)
        : id(i), name(n), value(v) {}
};

// 使用 ResourceManager 管理
auto& resourceManager = Resource::ResourceManagerModule::GetInstance();
resourceManager.Startup();

auto handle = resourceManager.Get<DataResource>(
    Resource::FromGenerator<DataResource>(
        "my_data",
        []() -> std::unique_ptr<DataResource> {
            return std::make_unique<DataResource>(1, "Resource1", 100.0f);
        }
    )
);

if (handle) {
    std::cout << "ID: " << handle->id << std::endl;
}

4.2 使用 FromPointer 接收外部对象

// 外部创建的对象
auto externalPtr = std::make_unique<ComplexResource>("data");

// 交给 ResourceManager 管理
auto handle = resourceManager.Get<ComplexResource>(
    Resource::FromPointer<ComplexResource>(
        "external_resource",
        std::move(externalPtr)
    )
);

4.3 自定义销毁函数

auto handle = resourceManager.Get<MyResource>(
    Resource::FromGenerator<MyResource>(
        "custom_resource",
        []() { return std::make_unique<MyResource>(); },
        [](MyResource* p) {
            // 自定义销毁逻辑
            std::cout << "Custom destroy" << std::endl;
            delete p;
        }
    )
);

4.4 零引用回调

auto handle = resourceManager.Get<MyResource>(
    Resource::FromGenerator<MyResource>(
        "callback_resource",
        []() { return std::make_unique<MyResource>(); },
        nullptr,  // 默认 delete
        [](const std::string& key, MyResource* res) {
            // 零引用时的回调
            std::cout << "Resource " << key << " released" << std::endl;
        }
    )
);

4.5 引用计数和句柄拷贝

auto handle1 = resourceManager.Get<MyResource>(...);
std::cout << "RefCount: " << handle1.GetRefCount() << std::endl;  // 输出：1

// 拷贝句柄 - 引用计数增加
auto handle2 = handle1;
std::cout << "RefCount: " << handle1.GetRefCount() << std::endl;  // 输出：2

// 再次获取同一资源
auto handle3 = resourceManager.Get<MyResource>(
    Resource::FromKey<MyResource>("my_key")
);
std::cout << "RefCount: " << handle1.GetRefCount() << std::endl;  // 输出：3

5. 工厂函数

工厂函数 用途 参数
FromGenerator<T> 从生成器函数创建资源 key, generator, destroyFunc, onZero, onRestore
FromPointer<T> 从已有指针创建资源 key, unique_ptr, destroyFunc, onZero, onRestore
FromConfig<T> 从配置文件创建资源 fileConfig, loader, onZero, onRestore
FromKey<T> 从已有资源恢复句柄 key

6. 完整测试示例

#include <iostream>
#include "resource_manager_module.h"

// 1. 定义简单资源类（无需继承）
struct DataResource {
    int id;
    std::string name;
    float value;
    
    DataResource(int i, const std::string& n, float v)
        : id(i), name(n), value(v) {}
    
    ~DataResource() {
        std::cout << "~DataResource" << std::endl;
    }
};

int main() {
    auto& rm = Resource::ResourceManagerModule::GetInstance();
    rm.Startup();
    
    // 2. 加载资源
    auto h1 = rm.Get<DataResource>(
        Resource::FromGenerator<DataResource>(
            "data1",
            []() { return std::make_unique<DataResource>(1, "Res1", 100.0f); }
        )
    );
    
    // 3. 使用资源
    if (h1) {
        std::cout << "id=" << h1->id 
                  << ", name=" << h1->name
                  << ", value=" << h1->value << std::endl;
    }
    
    // 4. 拷贝句柄
    auto h2 = h1;
    std::cout << "RefCount: " << h1.GetRefCount() << std::endl;
    
    // 5. 关闭模块
    rm.Shutdown();
    
    return 0;
}

7. 与旧版本的对比

特性 旧版本 新版本
资源类型要求 必须继承 ILoadable 任意类型
生命周期管理 通过 Release() 接口 通过 ControlBlock
销毁策略 固定的 Release() 调用 可自定义销毁函数
外部资源接入 需要包装为 ILoadable 直接通过 FromPointer

8. 注意事项

句柄拷贝语义：ResourceHandle 的拷贝构造和拷贝赋值会共享 ControlBlock，增加引用计数。这不是普通的值语义复制。
销毁函数：如果不指定销毁函数，默认使用 delete。对于特殊分配方式的资源（如内存池分配），请提供自定义销毁函数。
向后兼容：旧的 ILoadable 接口仍然保留，但已标记为弃用。现有代码无需修改即可继续使用。
